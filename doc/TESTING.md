# AI 数字看板平台 - 整体测试用例与操作指南

在开始测试之前，请确保后端（FastAPI）和前端（Vite）均已成功启动。
- **启动后端**：在 `backend` 目录下运行 `uvicorn app.main:app --reload`
- **启动前端**：在 `frontend` 目录下运行 `npm run dev`

本测试文档将指引你完成从“连接数据源”到“生成面板”并“导出脱机文件”的全流程测试。

---

## 测试阶段 1：数据源管理与连通性验证

### 用例 1.1：添加内置模拟数据源 (API)
1. 在侧边栏点击 **[项目列表]**，新建一个名为“年度汇报项目”的项目并进入。
2. 在左侧菜单点击 **[数据源]**，进入数据源管理页。
3. 点击 **[＋ 添加数据源]**，在“API 数据源”标签下，输入：
   - 名称：`模拟销售数据`
   - URL：`/api/mock/sales`
   - 方法：`GET`
4. 点击 **[🔍 探测]**，下方应显示绿色成功提示（发现多列字段）。
5. 点击 **[保存]**。

### 用例 1.2：连接 SQLite 数据库数据源
1. 继续在数据源管理页点击 **[＋ 添加数据源]**，切换到 **[🗄️ 数据库]** 标签。
2. 在下拉框中选择表名 `sales`（该表由初始化脚本生成）。
3. 自动带出 SQL 语句 `SELECT * FROM sales`。
4. 点击 **[🔍 探测表]**，应提示探测成功并显示行数。
5. （可选）修改 SQL 为 `SELECT category, sum(revenue) as revenue FROM sales GROUP BY category LIMIT 10`，再次探测。
6. 点击 **[保存]**，确认数据源列表中出现带有 📋 图标的数据库源。

## 测试阶段 2：素材库与 AI 模板生成

### 用例 2.1：浏览内置组件
1. 点击左侧菜单 **[素材库]**。
2. 确认页面下方正确分类展示了 15 种内置组件（KPI、折线图、仪表盘、滚动字幕等）。

### 用例 2.2：手动新建自定义模板
1. 点击右上角 **[＋ 新建模板]**。
2. 填写名称为“关键营收卡”，图标选“💰”，基础组件选 **[KPI 指标卡]**。
3. 在下方预设属性中，将标题改为“总营收”，颜色选择你喜欢的颜色。
4. 保存后，应在页面上方的“我的自定义模板”中看到新生成的卡片。

### 用例 2.3：AI 一句话生成专属模板 (亮点测试)
1. 在素材页中间的 **[AI 生成]** 输入框内，输入：“生成一个标题为今日订单数的数字翻牌，前缀加上总量，颜色改成金色”。
2. 点击 **[🤖 AI 生成]**。
3. 等待几秒后，上方模板列表中应自动出现一个带有 🤖 图标的新模板，点开编辑应能看到属性已被 AI 自动填好。

## 测试阶段 3：编辑器核心流与沉浸式操作

### 用例 3.1：创建看板与放置组件
1. 点击左侧菜单 **[看板]**，新建一个看板并命名为“大中华区销售看板”，点击进入编辑模式（全屏）。
2. 从左侧边栏的 **[⭐ 我的模板]** 中，点击刚才手动创建和 AI 创建的模板，它们应出现在画布上。
3. 从左侧基础组件中，点击一个“折线图”和一个“饼图”，在画布上拖拽并调整大小。
4. 验证：选中组件后，按 `Delete` 键能否删除？按 `Ctrl+Z` 能否撤销？按 `Ctrl+D` 能否复制？

### 用例 3.2：编辑器内侧边栏绑定数据 (沉浸式体验测试)
1. 点击顶部右侧的 **[⚡ 数据源]** 按钮，弹出右侧数据源抽屉。
2. 在抽屉内尝试点击“＋ 添加数据源”，测试是否可以直接在编辑器不丢失进度的前提下，新增一个 `users` 表的数据库数据源。
3. 选中画布上的“折线图”。
4. 在右侧 **[属性]** 面板的底部找到“数据源绑定”，选择刚刚添加的 `sales` 或 `users` 数据源。
5. 配置字段映射（如 x轴映射为 `month` 或 `date`，y轴映射为 `revenue` 或 `active_users`）。
6. 图表应立即渲染出真实数据。

### 用例 3.3：画布独立主题切换
1. 点击顶部右侧的主题切换下拉框（默认显示“🎨 跟随系统”）。
2. 尝试选择 `赛博霓虹 (cyber-neon)` 或 `深林秘境 (forest-green)`。
3. **重点验证**：画布区域的背景和卡片颜色应立即改变，但编辑器外壳（顶部工具栏、左右侧边栏）的颜色不应受到影响（保持原系统主题）。

### 用例 3.4：状态栏与保存
1. 观察编辑器最底部的状态栏，确认组件数和数据源数显示正确。
2. 按快捷键 `Ctrl+S` 或点击顶部 **[💾 保存]**。
3. 状态栏应更新显示“💾 刚才的保存时间”。

## 测试阶段 4：AI 智能副驾 (自然语言操作)

### 用例 4.1：自然语言排版与控制
1. 在编辑器右侧面板切换到 **[🤖 AI]** 标签。
2. 在输入框中向 AI 发送指令：“把所有的卡片对齐到画布左边，并把背景色改成深蓝色”。
3. 验证 AI 是否正确识别意图，并在画布上直接执行了组件属性和位置的修改。

### 用例 4.2：数据洞察与智能推荐
1. 在画布上选中已经绑定了数据的“折线图”。
2. 向 AI 发送指令：“帮我分析一下这个图表的数据走势”。
3. 验证 AI 是否能读取绑定在图表上的数据，并给出合理的文本总结与商业洞察。

---

## 测试阶段 5：脱机导出与独立部署

### 用例 5.1：生成独立 HTML 文件
1. 确保画布上已有绑定了数据源的图表，并设定了喜欢的主题。
2. 点击顶部栏右侧的 **[📦 导出]** 按钮。
3. 浏览器会自动下载一个以项目名命名的 `.html` 文件。

### 用例 5.2：脱机文件验证
1. 在本地资源管理器中，双击直接用浏览器打开刚下载的 `.html` 文件（无需启动开发环境）。
2. **验证以下几点**：
   - 布局和大小是否和编辑器中一致（会根据你的显示器自动缩放）。
   - 图表颜色是否继承了你设置的“画布独立主题”的色彩规范。
   - 绑定的 API 接口数据是否被正确渲染到图表中。
